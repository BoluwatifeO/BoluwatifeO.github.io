What is Information Technology security?

IT security guards against unauthorized access to company assets such as computers, networks, and data. It safeguards sensitive data's integrity and confidentiality by preventing sophisticated hackers from obtaining it.

Types of IT security

  • Internet security: Internet security includes the protection of information sent and received in browsers, as well as network security affecting web-based applications. These measures are designed to analyze incoming internet traffic for viruses and other malicious information. This protection can be provided by firewalls, antimalware, and antispyware software.
  • Network security: The purpose of network security is to keep unauthorized or malicious users out of the network. This ensures that the usability, dependability, and integrity of the system are not threatened. This level of protection is essential to prevent a hacker from accessing network data. It also prevents them from interfering with users' access to or use of the network. Network security has become increasingly difficult as organizations increase the number of endpoints and migrate services to the public cloud.
  • Cloud security: Because applications, data, and identities are shifting to the cloud, people are connecting directly to the Internet and are not protected by the traditional security stack. 6 Cloud security can help to protect software-as-a-service (SaaS) applications and public cloud usage. Cloud security can be achieved by utilizing a Cloud Access Security Broker (CASB), a Secure Internet Gateway (SIG), and cloud-based Unified Threat Management (UTM) (UTM).
  • Endpoint security: Devices are protected at the endpoint level via endpoint security. Endpoint security can safeguard devices such as cell phones, tablets, laptops, and desktop PCs. Endpoint security keeps devices from connecting to hostile networks that represent a risk to businesses. Advanced malware protection and device management tools are included in endpoint security.
  • Application security: To ensure that programs cannot be hacked, they are specifically coded to be the safest they can possibly be at the time of their construction. As an additional layer of protection, the software's code is analyzed to find any faults that may exist within it.

Why is Information Technology security important for businesses?

IT security ensures that applications implemented on an organization's information technology infrastructure run safely. This is because, in order to protect the data, the company will utilize or install appropriate software to secure the data, such as antivirus and other protected apps. To safeguard the programs that are installed and the data stored on computers, information security is essential in an organization. Aside from data protection, the application deployed must also be protected because it may cause data damage or injury. Information Technology security will protect the data collected and used by the organization. If the information is not protected, it can be accessed by anyone. If the information falls into the hands of the wrong people, it has the power to kill lives, destroy businesses, and wreak havoc. IT security programs will ensure that the organization's data is protected in accordance with corporate and legal standards by implementing measures to safeguard the data. In addition to protecting an organization's information, taking precautions to prevent identity theft is also a matter of protecting one's privacy.

IT security VS Cybersecurity

The similarities between IT security and cybersecurity:

Both terms refer to techniques that try to provide computer systems with security and protection against hostile activities and data breaches. Moreover, IT security and cybersecurity are closely related disciplines. They include comparable and complementary processes, but they must be distinguished to achieve an effective and successful application. IT security and cybersecurity both have a component that deals with the physical protection of data. Both techniques take various precautions to keep the information safe, such as locking the server room door and granting authorization to particular individuals. Furthermore, IT security and cybersecurity assess the worth of the data they seek to safeguard. In other words, both techniques strive to concentrate on the most critical facts. Assume, for example, that a person owns a bank. The list of client names is significant data, but the ID numbers, PIN codes, or addresses of these consumers are more important to secure. Most steps are made in both IT security and cybersecurity to secure the most sensitive data as best as possible.

Differences between IT security and cybersecurity:

The goal of IT security is to keep an organization's data secure and reliable. As the broad description implies, IT security encompasses a wide range of issues, including cybersecurity. That is, one can be an IT expert without specializing in cybersecurity.

Cybersecurity, like IT security, strives to keep information safe, but it focuses specifically on data in digital form: mobile devices, tablets, laptops, workstations, servers, networks, and so on. The goal of all cybersecurity practices is to prevent unwanted access to electronic data. To do this, cybersecurity specialists use various protocols and methods, including identifying sensitive and/or valuable data, the mitigation of vulnerable areas in an organization's security façade, risk assessment, and much more.

As previously stated, both IT security and cybersecurity seek to safeguard information. IT security encompasses a broader range of issues. It focuses on safeguarding critical data against all types of threats. Furthermore, it interacts with both digital and analogue information. While cybersecurity focuses on digital information, it also deals with other issues such as cybercrime, cyber assaults, cyberfraud, and law enforcement.

System Threats, Attacks and Vulnerabilities

Any probable occurrence, whether malicious or not, that has the potential to harm an asset is considered a threat. A threat, in other words, is any unfavourable occurrence that may occur to assets.

Any activity that exploits a vulnerability or poses a threat is considered an attack. Two types of attacks are sending malicious input to an application or flooding a network in an attempt to deny service.

A weakness that allows a danger to occur is known as vulnerability. This could be a result of poor design, inappropriate settings, or insecure coding techniques.

Differences between Threat and Attack

  • A threat and an assault differ primarily in that a threat can be either purposeful or inadvertent, but an attack is always intentional.
  • An attack is an attempt to cause damage, whereas a threat is a circumstance that has the potential to cause loss or harm.
  • A threat to the information system does not imply that information has been altered or damaged; but, if the attack is successful, there is a chance of altering, damaging, or gaining information.

Threats


Threats to network security are classified into four categories:
  1. Unstructured threats
    Unstructured threats frequently entail unfocused attacks on one or more network systems, which are frequently carried out by individuals with little or emerging abilities. The offender is most likely unaware of the systems being targeted and infected. These attacks are typically carried out by people who lack integrity and have an abundance of idle time on their hands. Malicious intent may or may not exist, but indifference to the harm done to others is always there. Innumerable sites on the Internet offer computer codes such as a Trojan horse or a virus that can be updated or copied in its entirety. In each case, these are small programs designed by humans. They are not alive and cannot evolve from anything. Some often used terms to be aware of are as follows:
    Virus In other words, a program that can reproduce itself with little or no user input, and the programs that are replicated also have the ability to replicate themselves.
    Worm A virus that replicates itself on other drives, systems, or networks in order to spread itself around the world. As long as an email system is used, a worm can send copies of itself to all of the addresses stored in the address book.
    Trojan horse An application that appears to be useful or pleasurable, such as a game or screensaver, but is actually conducting other tasks in the background, such as deleting or modifying data or recording passwords or keystrokes. Because it does not replicate itself, a true Trojan horse is not a virus.

    Unstructured attacks can quickly circle the globe in a matter of hours, damaging networks and individuals all over the world, because malware duplicates itself and sends a copy to everyone in the recipient's address book. A company's reputation can be damaged if word of an assault on its website spreads, or users' liberties can be curtailed when more stringent policies and practices are developed to defend against future attacks, even if the initial purpose was more thoughtless than malicious.

  2. Structured threats
    More targeted than unstructured attacks, structured threats come from those with higher-level knowledge who purposefully strive to compromise an entire system. Depending on how the system was discovered, it could have been a result of a random search, or it could have been deliberately picked. Assailants are frequently well-versed in hacking tools, network designs, and security protocols, and they are able to develop scripts or apps to accomplish their goals. Structured attacks are motivated by factors other than curiosity or peer competition. It's possible that the projects are motivated by greed, politics, racism (or any other form of intolerance), or law enforcement, among other factors. Incentives include identity theft and credit card information theft, as well as other crimes when the payout is not directly tied to the attack.
  3. Internal threats
    Individuals with or who have had permitted network access are the source of internal dangers. This might be a dissatisfied current employee, an opportunistic former employee, or a dissatisfied former employee with current access. Former network employees may still be using a hijacked or previously created account, even if the account is no longer active.
    Real-World Examples of Insider Threat-Induced Breach:
    Case 1: Employee negligence resulted in the leak of a Microsoft database.

    What transpired? A security researcher discovered a publicly available Microsoft customer support database with 250 million records accumulated over 14 years at the end of December 2019. Customer emails and IP addresses were stored in the database, as were the notes taken by Microsoft support personnel. The database was exposed to the public for nearly a month. Microsoft patched it the same day the problem was reported.

    What were the consequences? Due to the fact that the exposed data did not contain personally identifiable information, Microsoft was not fined or penalized, and the business took swift action to remedy the matter and warned users. When it came to Microsoft's data breach, it was lucky that it was found towards the tail end of 2019. The California Consumer Privacy Act went into effect a few days later, on January 3, 2020. Everyone who is harmed as a result of a violation faces a $750 fine under this law. Microsoft may have been penalized millions of dollars under the new legislation.

    What caused that to happen? A new version of the Azure security rules was issued by Microsoft at the beginning of December 2019. An unintentional leak was caused by Microsoft workers mishandling such restrictions. As a result, no password or two-factor authentication was used to protect access to the database. Furthermore, the company's detection time may have been greatly shortened by monitoring user data and analyzing conduct regarding sensitive assets.


    Case 2: Marriott's data was compromised as a result of a hacked third-party app.

    What transpired? An application used by Marriott to provide guest services was breached in January 2020 by hackers. The attackers were able to acquire access to 5.2 million customer records at Marriott. Contact information, gender, birthdates, loyalty account data, and personal preferences were all included in these records. After noticing suspicious activity in February 2020, Marriott's security staff resolved the insider-caused security issue.

    What were the consequences? Because the compromised material contains personally identifiable information, Marriott may face substantial consequences. In 2018, Marriott had to defend GDPR fines of £99 million (approximately $124 million) for a data breach.

    What caused that to happen? Using the credentials of two Marriott workers, the attackers got access to one of the hotel chain's third-party applications. For two months, Marriott's cybersecurity systems were unaware of any unusual behaviour on these employees' profiles. Marriott may have spotted the occurrence sooner if it had adopted third-party vendor monitoring and user and entity activity analytics.

  4. External threats
    Individuals or organizations working outside of a firm can pose external risks. They are not authorized to use the computer systems or network. They frequently connect to networks over the Internet or dial-up access servers.
    How can external threats manifest themselves?

    Malicious Software: A vast range of software is malicious software, and it is meant to harm a machine. Some are less harmful than others, such as adware, while others, such as viruses, can completely destroy a hard disk. Here are some examples of malware that we may encounter:

    • Spyware collects information about the user it has infected and discreetly sends it to third-party sources; this might be done through the use of keyloggers, which collect information from what you enter, such as passwords.
    • Ransomware prohibits people from accessing computer systems by encrypting storage media and demanding a fee to be paid in order to regain access.
    • Adware displays unwanted adverts in order to generate revenue, sometimes distracting the user by using pop-ups that do not close when dismissed.

    Hacking: Hacking is a broad phrase that refers to the use of weaknesses in a computer system to gain unauthorized access to the system and its data. The technique of assault is referred to as an "attack vector," and it frequently entails exploiting vulnerabilities in areas such as Wi-Fi, Bluetooth, internet connections, or getting internal network access.

    Sabotage: Sabotage is a broad phrase that refers to any behaviour that intentionally disrupts services, generally through the use of:

    • Denial of Service attacks (DoS).
    • Distributing Malware.
    • Physically destroy computer equipment.

    This can be done by individuals, terrorist organizations, businesses, or governments. However, unlike hacking, where the goal is more likely to be information gathering or irritation, sabotage is deliberately malicious, with the goal of causing damage. This is plainly done with the intention of interfering with the victim's ability to execute their normal functions. This can have disastrous consequences if suitable healing measures are not put in place by individuals who have been wounded.

    Social Engineering: The term "social engineering" refers to techniques used to dupe people into providing confidential and private information. This can then be used to commit identity and bank fraud, as well as acquire system access. Phishing emails are a common example of this, in which an email is sent out by a bot or someone impersonating someone else. This email will frequently ask the recipient to respond with sensitive information or to click on a link to a web page. This web page may then seek data from the user or install malware on the user's computer.

More examples of Attacks we may encounter

  1. Spoofing
    Forging data (such as an IP address or username) in order to impersonate a legitimate entity is known as spoofing. Numerous varieties of spoofing exist, including:
    • When an attacker forges an email's sending (From, or source) address, this is known as email spoofing
    • The technique by which an attacker alters the originating IP address in a network packet in order to disguise their identity or mimic another computing system is known as IP address spoofing.
    • When an attacker changes the Media Access Control (MAC) address of their network interface in order to conceal their identity or look like someone else, this is known as MAC spoofing.
    • When an attacker develops a fraudulent biometric sample in an attempt to mimic another user, this is known as biometric spoofing.
  2. Sniffing
    Packet sniffing occurs when a program records wired or wireless network packets destined for other hosts.
    • Everyone in the vicinity has access to wireless traffic.
    • Antennas can increase range by miles.
    • Depending on the network location, wired traffic is available.
    • ARP spoofing can divert traffic to a sniffing machine if the network location is undesirable.

    Sniffing is used to
    • Obtain passwords (FTP, IMAP, etc.)
    • Obtain other confidential information
  3. Man-in-the-middle
    A man-in-the-middle assault is an active eavesdropping attack in which the attacker connects to both parties and passes messages between them.
  4. Backdoor
    In a computer system, cryptosystem, or algorithm, a backdoor is any secret approach for circumventing standard authentication or security mechanisms. They can exist for a variety of causes, such as poor design or insufficient layout. They could have been added by an authorized individual to provide legitimate access, or by an attacker for malicious purposes; either way, their presence exposes a vulnerability. Backdoors can be difficult to detect, and they are usually discovered by someone who has access to the application source code or a thorough understanding of the computer's operating system.
  5. Phishing
    Phishing is the practice of deceiving users in order to get sensitive information such as usernames, passwords, and credit card information. Phishing is typically carried out through the use of email spoofing or instant messaging, and it frequently encourages users to enter information at a bogus website with a "look" and "feel" that is substantially comparable to the legitimate one. The false website frequently requests personal information, such as log-in information and passwords. This information can subsequently be utilized to acquire access to the individual's authentic account on the authentic website. Phishing is a form of social engineering in which the victim's confidence is exploited. Attackers are employing novel methods to get access to real-life accounts. Individuals are routinely sent bogus electronic bills claiming that they have purchased music, applications, or other things and requiring that they click on a link if the transactions were illegal.

Vulnerabilities

Vulnerabilities exist in all software:

  • Server: Databases, DNS, mail server software, web servers, web apps, and so on. Mobile: Mobile phone operating systems, mobile applications, and so on
  • PC: Microsoft Office, Adobe Reader, and web browsers
  • Embedded: printers, routers, switches, VoIP phones, automobiles, medical gadgets, televisions, and so on. Third-party software includes the following: web browser plugins, ad affiliate network JavaScript include files and mobile ad libraries
Security Vulnerabilities Types
  • Operating System Vulnerabilities: These are weaknesses in an operating system that hackers can exploit to gain access to or destroy an asset on which the OS is installed. Default superuser accounts, which may exist in some operating systems, and hidden backdoor apps are two instances.
  • Network Vulnerabilities: These are defects in the hardware or software of a network that allow an unauthorized party to infiltrate it. Two examples are insecure Wi-Fi access points and improperly configured firewalls.
  • Vulnerabilities in Process Controls: Certain process controls can introduce vulnerabilities (or a lack thereof). One example is the use of weak passwords (which may also fall under human vulnerabilities).
  • Human Vulnerabilities: In many cybersecurity architectures, the human element is the weakest link. Failures by users can easily expose sensitive data, establish vulnerable entry points for attackers, or disrupt the system.

    John the Ripper is one of the most popular password testing and breaking applications because it integrates multiple password crackers into a single package, autodetects password hash types, and provides a configurable cracker. A broad variety of encrypted password forms are supported, including DES, MD5, Blowfish, Kerberos AFS, and Windows 2000/XP 2003 LM hashes, to name a few. The capabilities of additional modules have been enhanced to support MD4-based password hashes as well as passwords stored in LDAP, MySQL, and other databases.

  2. NetStumbler: Users of Windows can obtain free security software. Locating open access points in a wireless network is an important technique for wardriving. 23 There is no source code for the application, and it is solely available for Windows. This may be tough for some people to accept. Security must be able to edit open-source code. Despite this, NetStumbler is quite popular due to its active WAP-seeking approach.     How NetStumbler Functions
    NetStumbler transmits a Probe Request frame at the default rate of once per second and then listens for any responding Probe Response data frames from nearby access points or ad-hoc networks. Simply said, the request is an electronic "Hello! Is there anyone there?”, whereas the Probe Response is the response to that enquiry. Its response includes information such as the wireless network name (SSID) and the Machine Access Code (MAC) numbers. The response is equal to saying in 802.11, "Oh, hello! I’m here! My MAC address is 00:00:00:00:00:00, and my SSID is MySSID.” NetStumbler logs the response and reports it to the user via the UI if the request is successful.

Legal Issues and Global Regulations

There are a lot of legal issues that come with cyber-attacks. There is no global standard for judging and punishing cyber crimes and cybercriminals, and local authorities are sometimes unable to take action because of a lack of laws under which to prosecute. Attributing cybercrime and cyberattacks is a major concern for all law enforcement agencies. "Computer infections spread from one country to another, from one jurisdiction to another - all across the world, taking advantage of our inability to conduct such global police operations. As a result, the Internet is as if someone [had] handed all of the world's online criminals free plane tickets." International Actions

Europe

The General Data Protection Regulation (EU) 2016/679 was adopted by the European Parliament and the Council of the European Union on April 14, 2016. GDPR, which went into force on May 25, 2018, protects all EU and EEA citizens' data and privacy (EEA). Data security must be embedded into all company procedures that handle personal data. A Data Protection Officer is also required by GDPR for certain firms (DPO).

National Actions

Teams of computer emergency response personnel To safeguard network security, most countries have their computer emergency response team.

  • China: On February 27, 2014, China's Central Leading Group for Internet Security and Informatization (Chinese: 中央网络安全和信息化领导小组) was founded. 24 This Chinese Communist Party Leading Small Group (LSG) is led by General Secretary Xi Jinping himself and is staffed by relevant Party and state decision-makers. The LSG was established to address China's previous cyberspace decision-making mechanisms' unclear policies and overlapping tasks. The LSG is in charge of network security and IT strategy policies in the economic, political, cultural, social, and military realms. This LSG also coordinates key policy activities in the international arena that promote Chinese government-favoured norms and standards while emphasizing the notion of national sovereignty in cyberspace.

Information Technology security Careers

  1. Network Administrator: Network administrators are IT professionals who have specialized knowledge of various IT resources but are primarily concerned with the installation, implementation, development, and deployment of networking systems. They are also in charge of administering those information systems and implementing several levels of cybersecurity into the systems to stop or prevent cyber-attacks.
  2. Information Security Analysts: Information Security analysts develop and implement counter-cyberattack methods to secure the data of their organizations. Because of the nature of their employment, these individuals play an important role in the public, commercial, and non-profit sectors, as they safeguard a company's sensitive and fundamental information. The purpose of all information security analysts is the same: to protect against and avoid cyberthreats. However, the precise scope of labour varies depending on the nature and size of an organization. For example, an online firm like LinkedIn needs data leak prevention, whereas working in cybersecurity for the government may entail developing and maintaining a secure network that cannot be accessed by third parties. An information security analyst's primary responsibilities include monitoring networks for security vulnerabilities, reducing the damage caused by cyberattacks, and developing and teaching personnel on security practices.
  3. Security Engineer: A security engineer guards against hackers’ unauthorized access to a company's systems and networks. These individuals may work for an IT firm that outsources its services to several organizations, or they may work as an in-house security engineers, overseeing the digital security of a single company
  4. Cybersecurity Analyst: A cybersecurity analyst guards a company's hardware, software, and networks against cybercriminals. The analyst's major responsibility is to thoroughly understand the company's IT architecture, monitor it at all times, and evaluate risks that could potentially breach the network. 25 The cybersecurity analyst is constantly looking for methods to improve the security of the company's network and secure sensitive information. The cybersecurity analyst is also in charge of:
    • Configuring Tools: This could take the shape of virus software, password protection software, or vulnerability management software. They will assess the company's requirements and employ these tools to safeguard its data.
    • Reporting: The analyst will describe the current state of the network and evaluate its strengths. Learning to read these reports is one of the abilities required. They will display what is well-protected and alert you if there is any odd network activity.
    • Assess Weaknesses: While no network is completely safe, the goal is to make it as secure as feasible. Part of the job entails constantly testing all company networks for flaws before bad actors or external threats may affect them.

Conclusion

In conclusion, it is extremely important for organizations to utilize IT security in order to guard against unauthorized access to company assets such as computers, networks, and data. IT security prevents sophisticated hackers from obtaining information by safeguarding sensitive data's integrity and confidentiality. IT security and Cybersecurity refer to techniques that try to provide computer systems with security and protection against hostile activities and data breaches. They include comparable and complementary processes, but they must be distinguished to achieve an effective and successful application. IT security encompasses a broader range of issues. While cybersecurity focuses on digital information, it also deals with other issues such as cybercrime, cyber assaults, cyberfraud, and law enforcement. Any probable occurrence, whether malicious or not, that has the potential to harm an asset is considered a threat. Any activity that exploits a vulnerability or poses a threat is considered an attack. Because these threats are evolving, dynamic multi-point security solutions are essential. Finally, IT security jobs are in the mainstream. IT security specialists are employed by industries such as health care, retail, manufacturing, and finance to secure critical information from cyber intrusions. There is a great demand for specialists in the field because of the frequency of cyberattacks.
